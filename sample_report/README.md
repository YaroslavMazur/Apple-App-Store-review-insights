# Sample report

A complete example of what the project produces for one app, in three formats: the prose product analysis in [`report.md`](./report.md), the visual UI walkthrough in [`ui-guide.md`](./ui-guide.md), and the raw JSON and CSV outputs of every API endpoint in [`data/`](./data/). Together these answer the question "what does this thing actually give you?" without requiring the reader to clone the repo and run it.

The data was generated by hitting the live Apple App Store on 2026-05-25 with app id `324684580` (Spotify) and country `us`, requesting the hundred most-recent reviews. Each file in `data/` is the response body of one of the public API endpoints, pretty-printed for readability — the API itself returns minified JSON.

## Files

- **`report.md`** — the written product analysis. Reads top-to-bottom as if a human analyst wrote it. Aggregates the metrics, calls out the three pain-point themes Spotify reviewers are complaining about, and explains what's worth doing about each one.
- **`ui-guide.md`** — a section-by-section walkthrough of the dashboard with placeholders for screenshots that live in [`screenshots/`](./screenshots/). Useful for understanding what each chart and card on the dashboard is for without needing to run the app.
- **`screenshots/`** — annotated dashboard screenshots referenced from `ui-guide.md`.
- **`data/collect.json`** — the full response body from `POST /api/v1/reviews/collect`. Contains the metrics, the sentiment breakdown, the seven discovered themes with their keywords and statistics, the three derived actionable insights, and the 2D coordinates plus theme assignment for every review (the `review_map` field). This is the single fattest file because it's everything in one shot.
- **`data/metrics.json`** — the response from `GET /api/v1/reviews/{app_id}/metrics?country=us`. Just rating statistics, no NLP output. The lightest endpoint.
- **`data/insights.json`** — the response from `GET /api/v1/reviews/{app_id}/insights?country=us`. The persisted `InsightsReport` with sentiment, themes, actionable insights, and the 2D review map. Same shape as the `insights` field inside `collect.json`.
- **`data/reviews.json`** — the response from `GET /api/v1/reviews/{app_id}/raw?country=us&format=json`. The hundred raw reviews exactly as fetched from the App Store, no NLP overlay.
- **`data/reviews.csv`** — the response from `GET /api/v1/reviews/{app_id}/raw?country=us&format=csv`. Same data as `reviews.json`, in a spreadsheet-friendly flat shape. This is what the dashboard's "Download CSV" button produces.

The JSON shapes are documented in the OpenAPI spec at `http://localhost:8000/docs` when the API is running locally, and they map one-to-one to the Pydantic models in `api/app/models/schemas.py`.
